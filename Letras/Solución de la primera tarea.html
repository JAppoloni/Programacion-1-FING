Solución de la primera tarea


En este documento se incluye una solución de la tarea y se destacan algunos errores que fueron observados. Errores como los que aquí se comentan serán tenidos en cuenta en la corrección de la segunda tarea.

Soluciones

En tarea.pas se encuentra un archivo con algunas posibles soluciones.

Para las dos funciones solicitadas se incluyen dos posibles versiones.

Notar que en las dos soluciones para la función Generadora la cantidad de comparaciones que se ejecutan es a lo sumo 3.

Para la función Longitud, las dos soluciones presentadas incluyen un ciclo que determina el resultado cuando se cumple ĺargo = limite o anterior = ultimo, o sea que la iteración depende de dos condiciones. En una de la versiones se repite la iteración hasta que se cumpla alguna de esas dos condiciones (estructura repeat). En la otra versión se itera mientras se cumpla ĺargo < limite y anterior <> ultimo (estructura while).

Errores

Los casos que se describen en esta sección se consideran errores aun si el resultado de la compilación y la ejecución es exitoso.

Errores generales

  • Que una función termine sin asignarle a su nombre el resultado.
  • Usar el nombre de una función como una variable en el cuerpo de su implementación.
  • Invocar una función sin asignar su resultado, como si fuera un procedimiento.
  • Definir funciones function f(a:...): tipo; donde tipo NO ES un tipo simple.
  • Usar break, exit, array of integer, setlength (arreglos dinámicos) que no son parte del Pascal dado en el curso.

Errores en Generadora

  • No tener en cuenta que puede haber números iguales. 
{ si a = b, a < c entonces el menor es a o b, y c es el mayor }
if a < b and a < c then menor := a
else if b < a and b < c then menor := b
else menor := c

  • Hacer el código para ordenar los dígitos de manera creciente y luego, en vez de usar ese resultado, repetir esencialmente el mismo código para ordenarlos de manera decreciente.

  • Construcciones aparatosas para obtener los dígitos en vez de usar MOD y DIV. Por ejemplo

c := 0
while n >= 100 do
  c := c + 1;
  n := n - 100
end;
  • Crear secuencia de IFs repitiendo condiciones sin usar las posibilidades del ELSE.

Errores en Longitud

  • Usar arreglos. No se sabe qué largo va a tener la secuencia que se va a generar y además no se necesita conservar los valores que integran la secuencia, por lo que es un error definir una estructura para guardarlos.

  • No usar dos condiciones para terminar el ciclo:

    • Iterar siempre hasta encontrar dos valores consecutivos iguales, sin controlar si el largo de la secuencia es igual a limite.

    • Iterar limite veces sin controlar si ya se encontraron dos consecutivos iguales.

  • Usar el conocimiento que se tiene del comportamiento, no especificado, de generadora (de esta generadora particular). Por ejemplo sabiendo que el siguiente de 495 es 495 controlar si el elemento es 495 en lugar de comparar con el siguiente.

  • Llamar a generadora dos o más veces con el mismo parámetro en lugar de llamar una sola vez y guardar su valor. 

while (largo < limite) and (actual <> generadora(actual)) do
begin
  ...
  actual := generadora(actual)
end;
  • Llamar a generadora una vez más (innecesariamente) cuando ya se pasó del límite.
while (largo <= limite) and  (anterior <> actual) do
begin
     ....
end;
if largo > limite then  longitud := -1
else  longitud := largo
  • Usar IFs para tratar casos especiales innecesariamente.
a := generadora(semilla);
if semilla = a then longitud := 1
else if limite = 1 then longitud := -1
else .....
